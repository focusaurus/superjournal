Backbone provides "MVC"

But it's really

  • Model
    • Also collections of models
  • Controller
    • backbone views are really more like controllers
    • backbone does NOT provide any view rendering capabilities directly
  • Router
    • backbone controller handles routing URL fragments Backbone is minimal and straightforward

Runs in the browser (primarily...)

Represents application business objects as models

Renders models in views

Provides REST/JSON CRUD operations to the server (Default)

Emits events to keep models and views synchronized

Can be customized for other transports, back ends, etc

What does backbone do for us?

  • When the page loads, it can fetch the collections of model
    • Makes HTTP GET for JSON data at the URL of each collection
  • Transforms raw JSON documents to backbone Model objects
  • Adds those objects to the browser-side backbone collection
  • Via event bindings, views are rendered with latest model data
  • When data changes, backbone provides a validation framework
  • If data is valid, backbone will make REST calls to the back end with the latest JSON data
  • Can refresh the views with changes made on the back end
